# <span data-ttu-id="f1d8a-103">Novidades da MED-V 2.0</span><span class="sxs-lookup"><span data-stu-id="f1d8a-103">What's New in MED-V 2.0</span></span>


<span data-ttu-id="f1d8a-104">A virtualização de área de trabalho do Microsoft Enterprise (MED-V) 2,0 evoluiu o suporte de compatibilidade do aplicativo para o Windows 7 e removeu a funcionalidade que não é necessária para esse cenário.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-104">Microsoft Enterprise Desktop Virtualization (MED-V) 2.0 has evolved the application compatibility support for Windows 7 and removed functionality that is not required for this scenario.</span></span> <span data-ttu-id="f1d8a-105">Por exemplo, recursos como criptografia do espaço de trabalho do MED-V, o servidor do MED-V centralizado e a transferência de corte do espaço de trabalho do MED-V foram removidos.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-105">For example, features such as encryption of the MED-V workspace, the centralized MED-V server, and MED-V workspace trim transfer have been removed.</span></span>

### <span data-ttu-id="f1d8a-108">Criação do espaço de trabalho do MED-V</span><span class="sxs-lookup"><span data-stu-id="f1d8a-108">MED-V Workspace Creation</span></span>

<span data-ttu-id="f1d8a-111">As funcionalidades de gerenciamento offline e transferência de corte foram removidas, além da funcionalidade de compactação e compactação do espaço de trabalho do MED-V proprietário.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-111">The offline management and trim transfer functionality were removed, in addition to the proprietary MED-V workspace encryption and compression functionality.</span></span> <span data-ttu-id="f1d8a-112">Quando você cria um espaço de trabalho do MED-V, um administrador deve preparar e configurar aplicativos e ferramentas de gerenciamento apropriados na imagem em vez de usar a ferramenta de preparação da máquina virtual fornecida no MED-V 1,0.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-112">When you create a MED-V workspace, an administrator should prepare and configure appropriate applications and management tools in the image instead of using the virtual machine preparation tool that is provided in MED-V 1.0.</span></span>

<span data-ttu-id="f1d8a-113">A execução do Sysprep na imagem do MED-V agora é necessária e validada durante a embalagem do espaço de trabalho do MED-V.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-113">Running Sysprep on the MED-V image is now required and validated during the packaging of the MED-V workspace.</span></span> <span data-ttu-id="f1d8a-114">O pacote do espaço de trabalho do MED-V fornece uma interface gráfica do usuário (GUI) que orienta o administrador por meio do processo de empacotamento.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-114">The MED-V Workspace Packager provides a graphical user interface (GUI) that guides the administrator through the packaging process.</span></span> <span data-ttu-id="f1d8a-115">O console do MED-V 1,0 foi removido junto com a funcionalidade de gerenciamento de imagens, o gerenciamento de perfis do espaço de trabalho do MED-V e a necessidade de testar e criptografar os espaços de trabalho do MED-V.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-115">The console from MED-V 1.0 was removed together with the functionality of managing images, managing MED-V workspace profiles, and the requirement to stage and encrypt MED-V workspaces.</span></span>

### <span data-ttu-id="f1d8a-116">Implantação do espaço de trabalho do MED-V</span><span class="sxs-lookup"><span data-stu-id="f1d8a-116">MED-V Workspace Deployment</span></span>

<span data-ttu-id="f1d8a-117">Para implantar um espaço de trabalho do MED-V, um administrador pode tirar proveito de suas ferramentas eletrônicas de distribuição de software.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-117">To deploy a MED-V workspace, an administrator is now able to take advantage of their electronic software distribution tools.</span></span> <span data-ttu-id="f1d8a-118">O método de recebimento de cliente disponível no MED-V 1,0 foi removido e o espaço de trabalho do MED-V agora é fornecido usando métodos fora do MED-V.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-118">The client-pull method available in MED-V 1.0 was removed and the MED-V workspace is now delivered by using methods outside MED-V.</span></span> <span data-ttu-id="f1d8a-119">Os administradores podem tratar os espaços de trabalho do MED-V como fariam com qualquer outro pacote de aplicativos e podem agendar implantações e instalações do MED-V usando suas ferramentas e processos existentes.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-119">Administrators can treat MED-V workspaces as they would any other application package and can schedule deployments and installations of MED-V by using their existing tools and processes.</span></span> <span data-ttu-id="f1d8a-120">As instalações do MED-V podem ser implantadas silenciosamente e podem ser facilmente gerenciadas dentro de uma infra-estrutura existente de distribuição de software.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-120">MED-V installations can be deployed silently and can easily be managed inside an existing software distribution infrastructure.</span></span>

### <span data-ttu-id="f1d8a-121">Gerenciamento do espaço de trabalho do MED-V</span><span class="sxs-lookup"><span data-stu-id="f1d8a-121">MED-V Workspace Management</span></span>

<span data-ttu-id="f1d8a-122">O espaço de trabalho do MED-V no MED-V 2,0 se baseia em um disco rígido virtual do Windows Virtual PC.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-122">The MED-V workspace in MED-V 2.0 is based on a Windows Virtual PC virtual hard disk.</span></span> <span data-ttu-id="f1d8a-123">O MED-V ampliou os recursos que o Windows Virtual PC oferece ao melhorar a experiência perfeita sem exigir criptografia ou ferramentas especiais para acessar o espaço de trabalho do MED-V.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-123">MED-V has extended the capabilities that Windows Virtual PC provides by improving the seamless experience without requiring encryption or special tools to access the MED-V workspace.</span></span>

<span data-ttu-id="f1d8a-124">Após a implantação do MED-V em uma estação de trabalho, o espaço de trabalho do MED-V pode ser aberto em modo de tela inteira usando o PC virtual do Windows.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-124">After MED-V is deployed to a workstation, the MED-V workspace can be opened in full-screen mode by using Windows Virtual PC.</span></span> <span data-ttu-id="f1d8a-125">Essa nova funcionalidade removeu a necessidade de políticas que definem uma preferência para modos de tela inteira ou contínua e também removemos a necessidade de forçar o diagnóstico e a solução de problemas em tela cheia.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-125">This new functionality removed the requirement for policies that set a preference for seamless or full-screen modes and also removed the need to force full-screen for diagnostics and troubleshooting.</span></span>

<span data-ttu-id="f1d8a-126">A publicação de aplicativos no espaço de trabalho do MED-V não é mais realizada com perfis e digitando manualmente o caminho para os aplicativos.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-126">Publishing applications to the MED-V workspace is no longer performed with profiles and by manually entering the path to applications.</span></span> <span data-ttu-id="f1d8a-127">Em vez disso, ocorre automaticamente à medida que os aplicativos são instalados no convidado.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-127">Instead, it occurs automatically as applications are installed on the guest.</span></span> <span data-ttu-id="f1d8a-128">O repositório de imagens central que incluiu versões das imagens que foram entregues por meio da transferência de corte foi removida.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-128">The central image repository that included versions of the images that were delivered through trim transfer is removed.</span></span> <span data-ttu-id="f1d8a-129">Em vez disso, o MED-V permite que os administradores gerenciem o espaço de trabalho do MED-V da mesma forma que seriam um computador físico, permitindo que aplicativos e atualizações sejam distribuídos sem a complexidade de uma infraestrutura MED-V dedicada.</span><span class="sxs-lookup"><span data-stu-id="f1d8a-129">Instead, MED-V enables administrators to manage the MED-V workspace as they would a physical computer, by letting applications and updates be distributed without the complexity of a dedicated MED-V infrastructure.</span></span>
