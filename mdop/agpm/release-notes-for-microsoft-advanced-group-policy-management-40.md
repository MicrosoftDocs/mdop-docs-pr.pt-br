## <span data-ttu-id="97b12-123">Problemas conhecidos com o AGPM 4,0</span><span class="sxs-lookup"><span data-stu-id="97b12-123">Known issues with AGPM 4.0</span></span>


### <span data-ttu-id="97b12-124">O comando importar da produção não importa configurações para um GPO com check-out</span><span class="sxs-lookup"><span data-stu-id="97b12-124">Import from Production command does not import settings into a GPO that is checked out</span></span>

<span data-ttu-id="97b12-125">Se você editar um GPO no ambiente de produção, deverá importar o GPO da produção para atualizar o GPO no arquivo morto offline.</span><span class="sxs-lookup"><span data-stu-id="97b12-125">If you edit a GPO in the production environment, you must import the GPO from production to update the GPO in the offline archive.</span></span> <span data-ttu-id="97b12-126">O comando **importar da produção** tem o objetivo de permitir que você execute um backup final da produção antes de terminar a edição para que você possa reverter para o backup de produção, se for necessário.</span><span class="sxs-lookup"><span data-stu-id="97b12-126">The **Import from Production** command is intended to let you perform a final production backup before you finish editing so that you can roll back to the production backup if it is necessary.</span></span>

<span data-ttu-id="97b12-127">Se o GPO estiver com check-out quando você executar o comando **importar da produção** , as alterações de produção não serão incorporadas à versão verificada do GPO.</span><span class="sxs-lookup"><span data-stu-id="97b12-127">If the GPO is checked out when you run the **Import from Production** command, then the production changes are not incorporated into the checked out version of the GPO.</span></span> <span data-ttu-id="97b12-128">No entanto, a versão importada do GPO é adicionada ao histórico do GPO, embora essa versão não esteja disponível para edição.</span><span class="sxs-lookup"><span data-stu-id="97b12-128">However, the imported version of the GPO is added to the history of the GPO even though that version is not available to be edited.</span></span> <span data-ttu-id="97b12-129">Quando o GPO tiver sido verificado, essa versão substituirá a versão importada no arquivo, mas ambos estarão disponíveis no histórico do GPO.</span><span class="sxs-lookup"><span data-stu-id="97b12-129">When the GPO is checked in, that version will supersede the imported version in the archive, but both are available in the history of the GPO.</span></span>

<span data-ttu-id="97b12-130">**Solução alternativa:** Verifique se o GPO está marcado antes de importá-lo da produção.</span><span class="sxs-lookup"><span data-stu-id="97b12-130">**Workaround:** Make sure that the GPO is checked in before you import it from production.</span></span> <span data-ttu-id="97b12-131">Se o GPO não tiver sido verificado antes de ser importado, você poderá usar o comando **Undo check out** para descartar suas alterações e reverter para a versão do GPO importada da produção.</span><span class="sxs-lookup"><span data-stu-id="97b12-131">If the GPO was not checked in before you imported it, you can use the **Undo Check Out** command to discard your changes and roll back to the version of the GPO that you imported from production.</span></span>

### <span data-ttu-id="97b12-132">Os GPOs com check-out não podem ser editados por vários minutos em um ambiente que usa uma topologia de vários sites do Active Directory</span><span class="sxs-lookup"><span data-stu-id="97b12-132">Checked out GPOs cannot be edited for several minutes in an environment that uses a multiple site Active Directory topology</span></span>

<span data-ttu-id="97b12-133">O AGPM usa um modelo cliente/servidor.</span><span class="sxs-lookup"><span data-stu-id="97b12-133">AGPM uses a client/server model.</span></span> <span data-ttu-id="97b12-134">O servidor do AGPM e o cliente do AGPM determinam seu próprio controlador de domínio mais próximo para operações de política de grupo.</span><span class="sxs-lookup"><span data-stu-id="97b12-134">The AGPM Server and the AGPM Client each determine their own closest domain controller for Group Policy operations.</span></span> <span data-ttu-id="97b12-135">Ao fazer o check-out de um GPO usando um cliente do AGPM, na verdade, ele é o servidor do AGPM que verifica o GPO do arquivo offline para uma pasta temporária na pasta SYSVOL.</span><span class="sxs-lookup"><span data-stu-id="97b12-135">When you check out a GPO by using an AGPM Client, it is actually the AGPM Server that checks the GPO out from the offline archive to a temporary folder in the SYSVOL folder.</span></span>

<span data-ttu-id="97b12-136">Se o servidor do AGPM e o cliente do AGPM estiverem em sites diferentes, o GPO com check-out temporário poderá não estar presente no controlador de domínio do site local por vários minutos ou até 30 minutos devido à latência de replicação do SYSVOL.</span><span class="sxs-lookup"><span data-stu-id="97b12-136">If the AGPM Server and the AGPM Client are in different sites, then the temporary checked out GPO may not be present on the local site's domain controller for several minutes or up to 30 minutes due to SYSVOL replication latency.</span></span> <span data-ttu-id="97b12-137">Nessa situação, você não pode editar o GPO com check-out usando o GPMC em um cliente do AGPM até que a duplicação de SYSVOL do GPO com check-out ocorra.</span><span class="sxs-lookup"><span data-stu-id="97b12-137">In this situation, you cannot edit the checked out GPO using the GPMC on an AGPM Client until SYSVOL replication of the checked out GPO has occurred.</span></span>

<span data-ttu-id="97b12-138">**Solução alternativa:** Como prática recomendada, você deve posicionar os clientes do AGPM no mesmo site que o servidor do AGPM ao qual se conectam para que você não precise aguardar que a replicação do SYSVOL ocorra antes de editar um GPO com check-out.</span><span class="sxs-lookup"><span data-stu-id="97b12-138">**Workaround:** As a best practice, you should position AGPM Clients in the same site as the AGPM Server to which they connect so that you do not have to wait for SYSVOL replication to occur before you can edit a checked out GPO.</span></span>

### <span data-ttu-id="97b12-139">O AGPM não poderá ler o limite de backup se a sua conta não tiver permissões para o arquivo morto</span><span class="sxs-lookup"><span data-stu-id="97b12-139">AGPM cannot read the backup limit if your account does not have permissions for the archive</span></span>

<span data-ttu-id="97b12-140">Em um cliente do AGPM, se você fizer logon usando uma conta que não tenha sido delegada para o arquivo do AGPM, inicie o console de gerenciamento de política de grupo (GPMC) e clique em **alterar controle**, você receberá o erro a seguir.</span><span class="sxs-lookup"><span data-stu-id="97b12-140">On an AGPM Client, if you log on by using an account that has not been delegated permissions to the AGPM archive, start the Group Policy Management Console (GPMC), and then click **Change Control**, you receive the following error.</span></span>

``` syntax
Failed to read backup purge limit for this domain. 

The following error occurred: 
You do not have sufficient permissions to perform this operation. 
Microsoft.Agpm.AccessDeniedException (80070005)
```

<span data-ttu-id="97b12-141">**Solução alternativa:** Entre em contato com um administrador do AGPM (controle total) e solicite que ele delegue o acesso ao AGPM para a sua conta.</span><span class="sxs-lookup"><span data-stu-id="97b12-141">**Workaround:** Contact an AGPM Administrator (Full Control) and request that they delegate access to AGPM for your account.</span></span> <span data-ttu-id="97b12-142">Se você for um administrador do AGPM, faça logon usando uma conta para a qual a função de administrador do AGPM seja atribuída para que você possa delegar o acesso à conta adicional.</span><span class="sxs-lookup"><span data-stu-id="97b12-142">If you are an AGPM Administrator, log on by using an account to which the AGPM Administrator role is assigned so that you can delegate access for the additional account.</span></span> <span data-ttu-id="97b12-143">Para obter mais informações, consulte "delegar o acesso no nível do domínio ao arquivo morto" na ajuda do AGPM.</span><span class="sxs-lookup"><span data-stu-id="97b12-143">For more information, see "Delegate Domain-Level Access to the Archive" in the AGPM Help.</span></span>
